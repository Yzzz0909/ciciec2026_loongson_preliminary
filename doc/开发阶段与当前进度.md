# 开发阶段与当前进度

这份文档把比赛任务、推荐开发顺序和当前项目进度合并到一起，方便直接拿来当推进看板。

## 1. 比赛任务可以先拆成两个大阶段

### 阶段一：基础 SoC 搭建

目标：

- 搭建 CPU + AXI + RAM + UART 的基础 SoC
- 在 Vivado 仿真中跑通 `hello_world`
- 在 FPGA 上看到串口输出 `Hello Loongarch32r!`

### 阶段二：中断与显示功能扩展

目标：

- 补全 `confreg`
- 加入外部中断控制器
- 接入 `DVI`
- 跑通 `int_test`
- 最终运行 `pinball_game`

## 2. 结合当前仓库，实际推进更适合分成 5 步

### 第 1 步：读懂工程

完成标志：

- 能解释 `CPU -> Axi_CDC -> AxiCrossbar -> RAM/UART`
- 能说清 `soc_top.v`、`mycpu_tb.v`、`create_project.tcl` 各自作用

### 第 2 步：把基础 SoC 骨架接起来

完成标志：

- Vivado 不再因为缺实例或连线问题而在 elaboration 阶段报错
- testbench 能正常找到 UART 实例

### 第 3 步：跑通启动代码

完成标志：

- `debug_wb_pc` 不再卡在启动阶段
- RAM 读写事务都能形成闭环

### 第 4 步：看到 UART 输出

完成标志：

- 看见 `Hello Loongarch32r!`
- 程序继续输出后续内容而不是打印一句后异常停止

### 第 5 步：扩展比赛功能

完成标志：

- 中断响应正确
- 显示输出正常
- 板上演示完整

- `DVI` 还没有接入并完成验证

## 4. 当前最关键的观察点

建议优先盯这几个点：

- `confreg` 的读写寄存器是否与软件期望一致
- `confreg_int` 是否能够正确产生中断请求
- `cpu_intrpt` 是否改为真实中断输入而不是固定常量
- `DVI` AXI 接口、地址映射和软件驱动是否一致
- `int_test` 是否能完整通过

## 5. 现阶段暂时不要分散精力的方向

在阶段二正式展开前，暂时不建议优先投入到这些方向：

- 上板后的细节优化
- 性能优化
- 风格性重构

1. 先把阶段一成功结果整理到 README / 阶段文档

当前可以把阶段一的仿真验收结论写成：

- `hello_world` 已在 Vivado 仿真中跑通
- 启动阶段已越过 `.data` 拷贝和 `.bss` 清零
- UART 已成功输出 `Hello Loongarch32r!`

- 当前未观察到异常跑飞现象

这意味着：

- `CPU + Axi_CDC + AxiCrossbar + RAM + UART` 主线已经贯通
