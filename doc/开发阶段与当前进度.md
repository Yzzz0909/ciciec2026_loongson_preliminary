# 开发阶段与当前进度

这份文档把比赛任务、推荐开发顺序和当前项目进度合并到一起，方便直接拿来当推进看板。

## 1. 比赛任务可以先拆成两个大阶段

### 阶段一：基础 SoC 搭建

目标：

- 搭建 CPU + AXI + RAM + UART 的基础 SoC
- 在 Vivado 仿真中跑通 `hello_world`
- 在 FPGA 上看到串口输出 `Hello Loongarch32r!`

### 阶段二：中断与显示功能扩展

目标：

- 补全 `confreg`
- 加入外部中断控制器
- 接入 `DVI`
- 跑通 `int_test`
- 最终运行 `pinball_game`

## 2. 结合当前仓库，实际推进更适合分成 5 步

### 第 1 步：读懂工程

完成标志：

- 能解释 `CPU -> Axi_CDC -> AxiCrossbar -> RAM/UART`
- 能说清 `soc_top.v`、`mycpu_tb.v`、`create_project.tcl` 各自作用

### 第 2 步：把基础 SoC 骨架接起来

完成标志：

- Vivado 不再因为缺实例或连线问题而在 elaboration 阶段报错
- testbench 能正常找到 UART 实例

### 第 3 步：跑通启动代码

完成标志：

- `debug_wb_pc` 不再卡在启动阶段
- RAM 读写事务都能形成闭环

### 第 4 步：看到 UART 输出

完成标志：

- 看见 `Hello Loongarch32r!`
- `test_end` 达到预期

### 第 5 步：扩展比赛功能

完成标志：

- 中断响应正确
- 显示输出正常
- 板上演示完整

## 3. 我们目前推进到哪里了

### 已完成

- 工程结构已经梳理清楚
- 基础 SoC 骨架已经补起来
- `soc_top.v` 中关键实例已经接上
- 之前 `u_axi_uart_controller` 缺失导致的 Vivado elaboration 报错已经修掉
- CPU 到 RAM 的读通路已经基本打通
- `Axi_CDC` 返回通道中的一部分 `X` 污染问题已经清理
- testbench、Vivado 工程、软件镜像加载链路已经能配合工作

### 正在进行中

当前主阶段是 `第 3 步：跑通启动代码`。

目前已经观察到：

- 程序能进入 `_start`
- 能经过 cache 初始化相关代码
- 能进入 `.data` 拷贝相关代码
- 局部波形里已经看到过写地址 / 写数据握手

当前卡点是：

- 程序还没有稳定越过启动阶段
- `debug_wb_pc` 会停在启动代码某些地址附近
- 还没有稳定进入 UART 初始化
- `uart_wen` 还没有真正持续工作

## 4. 当前最关键的观察点

建议优先盯这几个点：

- 看 `debug_wb_pc` 有没有稳定越过 `1c000104`、`1c000124`、`1c00012c`
- 看启动阶段的 RAM 读写事务是否真正形成闭环
- 先确认程序是否已经跑到 UART 初始化，再判断 `uart_wen` 是否异常

## 5. 现阶段暂时不要分散精力的方向

在 `hello_world` 还没稳定跑通之前，不建议优先投入到这些方向：

- `confreg`
- 外部中断控制器
- DVI
- `int_test`
- `pinball_game`
- 上板后的细节优化

## 6. 下一步最合理的顺序

1. 继续确认启动代码卡在哪一段
2. 把启动阶段的 RAM 读写事务彻底确认闭环
3. 让程序稳定进入 UART 初始化
4. 看到 `Hello Loongarch32r!`
5. 再开始补 `confreg`
6. 再做中断和 DVI
