# 上板前检查清单

这份清单只关注会影响 FPGA bit 流和板上行为的内容。

## 1. 当前结论

当前项目已经完成一次有效的板上验证：

- `pinball_game` 已能在 FPGA 板上实际运行

目前不需要为了上板恢复的内容：

- `sim/mycpu_tb.v`
  - 这是 testbench
  - 不参与综合
  - 不影响 bit 流
- Vivado 仿真波形脚本
  - 只影响仿真观察
  - 不影响板上行为

目前需要重点确认的内容：

- 最终软件目标是不是 `pinball_game`
- `pinball_game` 板上分支时间参数是否正确
- `confreg` / `DVI` / `cpu_intrpt` 当前 RTL 是否保持稳定版本
- Vivado 工程里是否没有混入仿真专用综合文件

## 2. 需要保留的代码状态

### 2.1 `pinball_game` 板上 timer 配置

文件：

- `sdk/software/examples/pinball_game/main.c`

当前代码：

```c
#define SIM_TIMER_CMP 5000
#define BOARD_TIMER_CMP 25000000

...

if (simu_flag)
{
    RegWrite(0xbf20f104, SIM_TIMER_CMP);
}
else
{
    RegWrite(0xbf20f104, BOARD_TIMER_CMP);
}
```

上板前要求：

- 保持 `BOARD_TIMER_CMP = 25000000`
- 不需要删除 `SIM_TIMER_CMP`
- 因为板上 `simu_flag = 0`，实际上会自动走板上分支

### 2.2 `int_test` 当前配置

文件：

- `sdk/software/examples/int_test/main.c`

当前代码：

```c
if (simu_flag)
{
    RegWrite(0xbf20f104, 5000); // timercmp 0.1ms
}
else
{
    RegWrite(0xbf20f104, 50000000); // timercmp 1s
}
```

说明：

- 如果这次上板目标是 `pinball_game`，这个文件不会影响最终 bit 流
- 所以它现在不用再改

### 2.3 `confreg` 当前状态

文件：

- `rtl/ip/confreg/confreg.v`

当前应保留的结论：

- `int_ctrl` 逻辑仍内嵌在 `confreg.v`
- `bit5` 已屏蔽，避免无关中断干扰
- 当前版本已经通过 `int_test` 验证

上板前要求：

- 不再继续重构 `int_ctrl`
- 保持当前稳定版本

## 3. Vivado 工程检查

上板前请确认：

- 目标顶层仍然是 `soc_top`
- 约束文件仍来自 `fpga/constraints`
- `rtl/ip/PLL_2019_2/clk_pll_sim_netlist.v` 没有被拿去综合

说明：

- 之前这个仿真 netlist 文件混入综合会报错
- 如果重新建工程后它又回来了，需要再手动从综合源里移除

## 4. 软件构建检查

建议顺序：

1. 进入 `sdk/software/examples/pinball_game`
2. 重新 `make`
3. 确认生成的镜像就是当前目标程序

建议确认点：

- 串口启动后应先看到：

```text
Please Choosetime !!
```

这可以作为“当前下载到板上的确实是 `pinball_game`”的第一判断。

## 5. 实际上板步骤

1. 重新 `make` 目标软件 `pinball_game`
2. 在 Vivado 中重新打开或刷新工程
3. 运行 `Synthesis`
4. 运行 `Implementation`
5. 生成 `bitstream`
6. 下载到 FPGA
7. 连接串口和显示器
8. 验证按键、显示、数码管、LED

## 6. 板上预期现象

理想情况：

- 串口先打印：

```text
Please Choosetime !!
```

- 按键可调整时间
- 确认键可进入游戏
- 显示器有图像输出
- 数码管和 LED 有响应

- [ ] 当前目标软件是 `pinball_game`

- [ ] 当前 RTL 保持稳定版本，不再额外重构
- [ ] `sim/mycpu_tb.v` 未参与综合
- [ ] Vivado 工程未混入 `clk_pll_sim_netlist.v`
